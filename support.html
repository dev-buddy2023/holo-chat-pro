🚀 Getting Started

🚀 快速上手

Holo Chat Pro lets you chat with any AI model through OpenAI-compatible APIs, with 31+ built-in tools at your command.

Holo Chat Pro 让您通过 OpenAI 兼容的 API 与任何 AI 模型聊天，并拥有 31+ 个内置工具。

Step 1: Get an API Key

步骤 1：获取 API Key

You'll need an API key from an OpenAI-compatible provider. Popular options:

您需要从 OpenAI 兼容的提供商获取 API 密钥。常见选择：

  • OpenRouter — access Claude, Gemini, DeepSeek, and 200+ models
  • OpenAI — GPT-4o, GPT-4o Mini
  • Any OpenAI-compatible provider or local LLM endpoint

Step 2: Add Your Key

步骤 2：添加密钥

Go to Settings → API Key and paste your key. It's stored securely in iOS Keychain — never shared or uploaded anywhere.

前往设置 → API Key 粘贴您的密钥。密钥安全存储在 iOS Keychain 中，绝不外传。

Step 3: Start Chatting

步骤 3：开始聊天

That's it! Choose a model, type or speak your message, and your AI can use 31+ tools to help you get things done.

完成！选择模型，输入或说出您的需求，AI 可调用 31+ 个工具帮您完成任务。

❓ FAQ

❓ 常见问题

Do I need my own API key? 需要自己的 API Key 吗？

Yes. Holo Chat Pro does not provide API services. You bring your own key and connect directly to your chosen provider. You are responsible for its availability and usage.

需要。Holo Chat Pro 不提供 API 服务。您自带密钥并直接连接您选择的提供商，自行负责其可用性和使用。

Is there a subscription? 需要订阅吗？

No. The app is free to download. You only pay your provider directly for the tokens you use. No markups, no hidden fees.

不需要。应用免费下载，您只需直接向提供商支付使用的 token 费用。无加价、无隐藏费用。

How do I switch between models? 如何切换模型？

Save multiple API profiles and switch with one tap. Keep different keys and model preferences organized.

保存多个 API 配置文件，一键切换。管理不同的密钥和模型偏好。

Where is my data stored? 数据存储在哪里？

Everything stays on your device. Chats are saved as individual files you can browse, back up, or transfer through the Files app. No data is sent to any server except your chosen API provider.

所有数据都保存在您的设备上。聊天记录以独立文件存储，您可以通过"文件"应用浏览、备份或传输。除了您选择的 API 提供商外，不会向任何服务器发送数据。

What models are supported? 支持哪些模型？

Any model available through your API provider. With OpenRouter you get 200+ models: GPT-4o, Claude 3.5/3.7, Gemini 2.0, DeepSeek V3/R1, Qwen, GLM, Groq, Mistral, Llama 3, and more.

您的 API 提供商提供的任何模型。使用 OpenRouter 可获取 200+ 模型：GPT-4o、Claude 3.5/3.7、Gemini 2.0、DeepSeek V3/R1、Qwen、GLM、Groq、Mistral、Llama 3 等。

What is prompt caching? 什么是 Prompt 缓存？

Some providers (like OpenRouter with Claude) support prompt caching, reusing repeated context from previous turns. This can cut your API costs by 50% or more on long conversations.

部分提供商（如 OpenRouter 的 Claude）支持 prompt 缓存，可重复使用之前对话的上下文，在长对话中可节省 50% 以上的 API 费用。

Can I link a project folder? 可以关联项目文件夹吗？

Yes. Link a conversation to a local project folder. The AI can read, write, search, and manage your project files — like a coding assistant built into your chat.

可以。将对话关联到本地项目文件夹。AI 可以读取、写入、搜索和管理您的项目文件——就像内置在聊天中的编程助手。

How do I recover deleted chats? 如何恢复已删除的聊天？

Deleted conversations go to the Recycle Bin and stay there for 30 days. You can recover or permanently delete them at any time.

已删除的对话会进入回收站，保留 30 天。您可以随时恢复或永久删除。

🔧 Troubleshooting

🔧 故障排除

"Invalid API Key" or "401 Unauthorized"

"Invalid API Key" 或 "401 Unauthorized"

  • Double-check your key in Settings → API Key
  • 检查设置 → API Key 中的密钥
  • Verify the key is active and has credits on your provider's dashboard
  • 在提供商的仪表盘中确认密钥有效且有余额
  • Ensure you're using the correct provider URL in your profile settings
  • 确保在配置文件设置中使用了正确的提供商 URL

No models showing up

没有显示模型

  • Check your internet connection
  • 检查网络连接
  • Your provider may be experiencing temporary downtime
  • 您的提供商可能暂时不可用
  • Try refreshing the model list in the WebUI or app settings
  • 尝试在 WebUI 或应用设置中刷新模型列表

Tools not working

工具不起作用

  • Make sure you're using a model that supports tool/function calling (e.g., Claude, GPT-4o, Gemini)
  • 确保您使用的模型支持工具/函数调用（如 Claude、GPT-4o、Gemini）
  • Some models have limited tool capabilities — check the model card in the browser
  • 部分模型工具能力有限——请在模型浏览器中查看模型卡片

App crashes or freezes

应用崩溃或无响应

  • Close the app and reopen
  • 关闭应用重新打开
  • Update to the latest version from the App Store
  • 从 App Store 更新到最新版本
  • If the issue persists, email us at dev_buddy2023@163.com
  • 如果问题仍然存在，请发送邮件至 dev_buddy2023@163.com
